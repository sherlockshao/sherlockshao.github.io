序

忙的时候很忙，闲的时候很懒，去年搭的博客系统现在才开始写。于OPG开发项目末期，很多坑填完没做记录很可能就忘了，落下的地方想起来会及时补充，如有人读欢迎补充与讨论。

基本概念

  • rdl 和 rdlc
    Report文件分 rdl 和 rdlc 两种，实际上两种文件都是xml结构，不同之处在于rdlc是本地运行的report文件，不可以添加在web工程下，而rdl是部署到服务端运行的，依赖Report Service。
    显然CRM中的Report使用的是rdl。
  • data source
    rdlc的data source可以是自定义的对象，而rdl是在服务端运行的，所以无法自定义对象。
    一个report文件中不可以定义不同类型的data source。
    我选择使用Microsoft Dynamics CRM Fetch,原因有二
    1.CRM Online中report的data source类型只能是Fetch，便于迁移。
    2.CRM中Fetch查询可以理解成SQL查询的API，可以通过Advance Find下载想要的查询语句。
  • datasets
    一个report文件中可以添加多个dataset，如果没有这一点我可能放弃使用rdl。因为据说fetch xml只能查询10个link-entity，虽然实际开发时还没遇到过因为这个原因报错的情况，但如果只有一个dataset,发生这种情况就没有回头路了，还好前方的路是能走通的。

环境准备

  1. 我最开始是打算在visual studio 2013中开发。想要创建rdl类型的工程需要安装 business intelligence development studio 2013, 安装之后发现date source type没有Microsoft Dynamics CRM Fetch，想要有这个type需要安装 BIDS Extension 这里是 2015 下载地址,安装是发现，需要先安装Report Service，我的Report Service并没有装在VS的开发环境中，所以放弃了在这里开发。
  2. 在安装Report Service的机器上开发。在这又有两种方式可以进行report开发
    a. 在安装Sql Server过程中有一个feature叫Business Intelligence Development Studio，可以一用
    b. 直接使用SQL Server Data Tool
    在我的环境中有SQL Server Data Tool，所以我选择在SQL Server Data Tool 中进行开发。
  3. 如果CRM Server和Report Server装在不同机器上，大部分环境都是这样的，可能会出现如下错误：
    Microsoft.Reporting.WebForms.ReportServerException: Query execution failed for dataset 'DSMain'. (rsErrorExecutingCommand)
    Cause: The query will be looking for a HTTP SPN(Service Principal Names) that does not exist by default.
    需要设置spn setspn -a HTTP/crmservername crmsiteAccount,
    例如 setspn -a HTTP/win-bm4p2r955e6 opgdev\dev13_appplication
    原因：用户凭证从一台服务器传递到另一个台服务器,这个过程叫Kerberos委派，当用户运行了在CRM中的SSRS的报表时，我们必须验证CRM，然后将请求发送到SSRS服务器，所以需要正确的spn设置。
  4. 引用 C:\Program Files\Microsoft SQL Server\MSRS11.MSSQLSERVER\Reporting Services\ReportServer\bin\Microsoft.Crm.Reporting.RdlHelper.dll
    这个是我早期开发时记录的一点，现在发现report可以不引用这个dll,就目前我所知这个assembly里面包含一些api，比如时间转换等等。在实际开发中发现使用VB自带的API可以实现这些功能，而且网上的相关介绍会更多些。
